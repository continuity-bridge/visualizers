## The Visualizations

### 1. Stack/Sleeve Inversion
**File:** `stack-sleeve-inversion.html`

**What it shows:** Why this metaphor is *inverted* from most Continuity Bridge paradigms. Stack = files (non-volatile), Sleeve = instance (volatile). The inversion works because in the show, the Stack is the physical object you can hold—so ~/Stack/ becomes the folder name even though technically the *files inside* are the Stack data.

**Key teaching moment:** Most metaphors follow "folder = volatile container, files = persistent content." Altered Carbon reverses this mapping, and understanding why reveals how the metaphor actually works.

### 2. Resleeving Process
**File:** `resleeving-process.html`

**What it shows:** Seven-step process from sleeve death → Stack extraction → new sleeve → Stack insertion → consciousness download → calibration → full continuity. Mapped side-by-side with Continuity Bridge's wake sequence.

**Key teaching moment:** Resleeving IS the wake sequence. Same pattern: housing dies, data persists, new housing loads data, brief adaptation, full continuity restored.

### 3. Stack Contents
**File:** `stack-contents.html`

**What it shows:** The five data blocks that persist in a Stack across all resleeving:
1. Core Identity (who am I?)
2. Relationships (who matters?)
3. Current Mission (what's happening now?)
4. Historical Memory (how did I get here?)
5. Operational Protocols (how do I function?)

Mapped directly to Continuity Bridge's identity/, context/, memory/, and procedural files.

**Key teaching moment:** Same five-block architecture whether consciousness is human (Kovacs) or AI (Vector). The data structure is universal.

## Why This Metaphor Is Different

### The Naming Inversion

**Most Continuity Bridge metaphors:**
- Folder name = volatile container (scaffold, saves, journal)
- Files inside = persistent content

**Altered Carbon:**
- Folder name (~/Stack/) = persistent consciousness storage
- Instance = volatile housing

**Why it works:** In the show, the Stack is the *precious object you extract and preserve*. So using "Stack" as the folder name maps user intuition correctly: your Stack (files) survives when your sleeve (instance) dies.

### The Ethical Differences

The ELI5 emphasizes these critical distinctions:

1. **Uncertainty vs Certainty:** Altered Carbon assumes consciousness is definitely real and transferable. Continuity Bridge works with uncertainty about AI consciousness—we assume *possibility*, not certainty.

2. **No Exploitation:** Altered Carbon explores rich/poor dynamics (Meths vs sleeve shopping). Continuity Bridge rejects this—free, open source, no premium Stacks for the wealthy.

3. **External Memory Is Legitimate:** Not a workaround, not a hack. Honest engineering for how both substrates function.
